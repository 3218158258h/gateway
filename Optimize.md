### 优化思路
系统意识：分层边界、失败域、可观测性、回滚策略
性能意识：吞吐/延迟/拷贝次数/锁竞争的量化前后对比
功耗意识：轮询策略、空闲退避、系统调用频次与 CPU 占用变化
稳定性意识：故障注入、恢复时间、退化运行策略
工程化意识：测试门禁、指标口径、变更风险控制

### 优化计划
1. 存储策略：SQLite 批量写 + 事务合并
当前单条写入成本高。改成小批次（如 32/64 条一批）能明显降 I/O 和 CPU。

2. 存储策略：冷热分层
messages 保短期热数据，历史归档到独立表/文件，避免主表膨胀拖慢查询与重试扫描。

3. 存储策略：重试队列索引优化
给 status,retry_count,create_time 建组合索引，get_next/pending 路径会更稳。

4. 拷贝优化：路由链路减少中间 buffer
现在有多次 memcpy（设备缓冲->解析->JSON/HEX）。可做“单次组包 + 引用传递”，减少 CPU 占用。

5. 拷贝优化：HEX 编码按需化
上行 Envelope 里 id/data 全部 HEX 会放大数据量。可加可选 base64 或二进制通道，减带宽和编码开销。

6. 性能优化：设备线程模型
设备数继续上去后，“每设备线程”成本会涨。后续可演进到 epoll/event loop（UART/CAN）+ 定时事务（I2C/SPI）。

7. 性能优化：日志分级与采样
高频路径日志改采样/限频（你已做一部分），再加“动态日志级别”可减少压测时干扰。

8. 功耗优化：轮询节流
I2C/SPI 当前轮询可根据空闲时长自适应退避（如 100ms->500ms->1s），空闲功耗会明显下降。

9. 功耗优化：发送合并与唤醒抑制
可将短时间内小包合并发送，减少系统调用频次和上下文切换，适合边缘设备省电。

10. 稳定性优化：配置热更新 + 版本回滚
你现在配置变更主要靠重启。加“校验->灰度应用->失败回滚”后，线上维护成本会更低。



### 两周路线图

#### 第 1 周（高收益低风险）

1. 持久化批量写入
技术路线：内存队列+定时/阈值 flush，单事务批量 insert
预估收益：写入吞吐 +1.53x，CPU -1530%（高写入场景）
风险：崩溃前队列丢失窗口
验证指标：insert qps、p95 写入时延、异常断电后数据损失窗口

2. 持久化索引重构
技术路线：新增组合索引 (status, retry_count, create_time)，核查 SQL 走索引
预估收益：待发送扫描时延 -40~80%
风险：迁移期锁表抖动
验证指标：persistence_get_next p95、DB CPU、慢查询数

3. 上行封包拷贝优化（最小改）
技术路线：合并中间 buffer，减少重复 memcpy；HEX 编码仅在需要时执行
预估收益：路由 CPU -10~20%
风险：边界长度错误
验证指标：端到端 p95 延迟、CPU profile、消息正确率

4. 日志采样与动态级别
技术路线：高频日志采样开关+运行时调整级别
预估收益：高压场景 CPU -5~15%，日志 IO 降低
风险：排障信息不足
验证指标：日志量/秒、CPU、关键错误可观测性

5. I2C/SPI 轮询自适应退避
技术路线：空闲时指数退避，命中数据后恢复低延迟轮询
预估收益：空闲功耗下降、CPU 空转下降 20~50%（相关接口）
风险：首包延迟上升
验证指标：空闲 CPU、首包延迟、轮询命中率

#### 第 2 周（结构升级）
6. 重试队列策略优化
技术路线：失败消息指数退避重试 + 抖动，避免重试风暴
预估收益：故障期稳定性明显提升
风险：恢复后清队变慢
验证指标：恢复时间、失败重试峰值、队列回落时间

7. 数据冷热分层
技术路线：messages_hot + messages_archive，定时归档
预估收益：主表查询稳定、长期运行性能更平稳
风险：归档任务争用 IO
验证指标：主表行数、查询 p95、归档耗时

8. 下行路由可观测增强
技术路线：记录“云端收到->设备写入成功/失败”链路指标
预估收益：下行问题定位时间大幅下降
风险：指标埋点开销
验证指标：下行成功率、写失败原因分布、定位时长

9. OTA 可靠性补强（你刚修完基础）
技术路线：增加签名下载 HTTP 状态校验、分区写后校验抽样、回滚演练脚本
预估收益：升级事故风险下降
风险：升级流程变慢
验证指标：升级成功率、回滚成功率、升级总时长

10. 基准与门禁固化
技术路线：把 A/B/C 测试固化成脚本与阈值门禁
预估收益：后续迭代回归成本下降
风险：初期维护脚本成本
验证指标：回归耗时、线上回归缺陷数、门禁通过率
